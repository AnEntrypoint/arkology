# Resident Management System

A Python application for managing community resident records, financial tracking, and demographic statistics.

## Features

- **Resident Registry**: Add, update, delete, and search resident records
- **Financial Tracking**: Monitor payments, taxes, charity allocations, and property values
- **Demographics**: Track age, gender, birth year, and other resident attributes
- **Community Statistics**: Automated reporting on living residents, voting eligibility, gender distribution
- **Status Tracking**: Monitor resident status (alive/deceased, voting eligibility, jury duty, disabilities)
- **Data Persistence**: All data stored in JSON format for easy integration

## Installation

No external dependencies required. Uses Python 3 standard library only.

```bash
python3 app.py
```

## Usage

### List all residents
```bash
python3 app.py list
```

Displays a formatted table of all residents with ID, name, age, payment, and tax information.

### View community statistics
```bash
python3 app.py stats
```

Generates a comprehensive report including:
- Total living residents
- Voting age residents
- Gender breakdown
- Total charity allocated
- Total payments and taxes

### Search for residents
```bash
python3 app.py search <name>
```

Search by last name, first name, or email. Example:
```bash
python3 app.py search Tea
```

### Export data
```bash
python3 app.py export [filename]
```

Exports all resident records to JSON. Default filename is `residents_export.json`.

## Data Structure

Each resident record contains:
- **Identity**: res_id, first_name, last_name, email, address
- **Demographics**: year_born, gender, age
- **Financial**: payment, net_tax, charity, property, wealth
- **Status**: alive, voting, jury_duty, disabled
- **Additional**: age_ubi, children, sunlight

## Data Files

- `residents.json`: Primary data store containing all resident records
- `residents_export.json`: Exported data (generated by export command)

## Architecture

The system is built with:
- **Resident class**: Data model for individual residents
- **ResidentRegistry**: Manages resident collection and persistence
- **Statistics**: Generates community reports and metrics
- **CLI**: Command-line interface for user interaction

All code is contained in a single `app.py` file for simplicity and portability.

## Extension

To add new functionality:

1. Add properties to the Resident class __init__ method
2. Update the ResidentRegistry CRUD methods if needed
3. Add new methods to Statistics class for metrics
4. Add new commands to the CLI class
5. Update the command parser in the main block
